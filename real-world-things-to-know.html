Real World

Things to know to make useful projects

Posted by John on August 24, 2019

A place we all find ourselves in along our learning journey is trying to get to a point where we can actually write useful programs, those beyond little scripts that change file names, or print things to the terminal screen. It's hard to continue learning if you aren't sure what prjects to do, and while other peoples lists of beginner or intermediate projects are useful, its the ones that directly impact your own world or your current job, that have you searching for solutions and overcoming problems. And thats what learning is.

Here are a few things that I leanred myself that helped me understand how to write useful programs pertinent to me, and kept me moving forward on my learning journey.

API's and how to integrate

If you intend to go anywhere near the web then getting to grips with how an API works is essential. Learning how connect and extract data using Python is a great way to utilise an API and transform your programs

An API is basically how programs talk to each other. We can send a request (the simplest being a GET request) over HTTP and request information, and get sent data back. Think about a weather API. We could send a request for the weather in London now, and recieve back that information instantly. Or we could request the weather for tomorrow, and so on. This might not be useful for us as is, but you could combine that information with something else and produce a result useful to you in someway.

There are hundreds of APIs available to you, lots free and some not. There's a music lyrics API, a Chuck Norris quotes API, Country information and even some games like Heartstone have an API to extract data.

JSON data

The data standard for API and web data is JSON, making this a vital thing to learn to work with. You can use the build in JSON Python library to parse the information and extract what you need.
